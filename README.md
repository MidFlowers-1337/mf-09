# 💒 婚礼协作小站

把每场婚礼的 **人、任务、钱** 拢到一处的协作工具。不用再在五六个微信群里翻信息，再也不会漏掉订花这种事。

## 快速开始

```bash
npm install
npm run dev
```

然后浏览器打开：**http://localhost:5173/**

> 数据库文件会自动生成在 `data/wedding.db`，删掉它就相当于重置。

---

## 功能总览

| 模块 | 能干什么 |
|------|---------|
| **婚礼概览** | 新人信息、日期、预算、倒计时、一眼看整体进度 |
| **任务看板** | 三列看板（待办/进行中/完成），设负责人、截止日、前置依赖 |
| **宾客管理** | 分组（男方/女方亲戚、朋友、同事...），桌次安排，统计人数防加桌乱 |
| **供应商** | 酒店、花艺、摄影、化妆、餐饮... 记合同金额和已付款 |
| **预算支出** | 每笔记账，跟预算总额实时比，超了自动标红提醒 |

## 手把手操作步骤

### 第 1 步：建一场婚礼

1. 打开 http://localhost:5173/ ，点击右上角 **「新建婚礼」**
2. 填：
   - 新郎 / 新娘姓名
   - 婚礼日期（倒计时会按这个算）
   - 预算总额（比如 100000）
   - 场地（可选）
3. 点 **「创建婚礼」**，就会进到婚礼概览页，自动建 5 个常用分组（男方亲戚、女方亲戚、朋友、同事、其他）

### 第 2 步：拆任务 + 设依赖

1. 进入婚礼后点顶栏 **「任务看板」**
2. 点右上角 **「新建任务」**，常见任务例子：

   | 任务 | 前置依赖 | 负责人 |
   |------|---------|-------|
   | 挑选酒店 | 无 | 策划师 |
   | 签酒店合同 | 「挑选酒店」完成 | 策划师 |
   | 试妆 | 无 | 新娘 |
   | 确定伴娘伴郎 | 无 | 新人 |
   | 设计请柬 | 无 | 设计师 |
   | 印刷请柬 | 「设计请柬」完成 | 设计师 |
   | 发送请柬 | 「印刷请柬」完成 | 新人 |
   | 预订婚车 | 无 | 新郎家长 |
   | 女方订花 | 试妆确认后 | 花艺师 |

3. **怎么设依赖**：
   - 新建任务时，直接在底部勾选已有任务作为前置
   - 已建好的任务，点卡片里的 **「管理依赖」** 可以随时勾选/取消
4. **依赖会自动锁任务**：
   - 前置没做完，按钮是灰的，点「进行」或「完成」会弹窗告诉你还缺什么
   - 防止漏了"印请柬"就直接"发请柬"这种事

### 第 3 步：录宾客，分组，排座位

1. 顶栏点 **「宾客管理」**
2. 左侧 **分组** 区：
   - 点快捷按钮加常用分组（男方亲戚、女方亲戚、朋友、同事、其他）
   - 点 **「+ 新分组」** 自定义（比如 "新娘同事"、"发小"）
3. 点右上角 **「添加宾客」**：
   - 姓名、电话、分组、桌次、到场状态（待确认/已确认/无法到场）、携伴人数
4. 左侧 **桌次** 区：
   - 点 **「+ 新桌」** 设桌号、容量（默认 10 人）、桌名（如「主桌」、「同事桌」）
   - 每桌显示 `已入座/容量`，坐满会打「坐满」标签，超员会打「超员」红标
5. **顶部统计条** 实时显示：
   - 总邀请（含携伴）
   - 已确认到场人数
   - 席位总容量 / 未安排入座人数 → **别搞错席数，酒店现场不会乱**
   - 待确认 / 婉拒

### 第 4 步：记供应商

1. 顶栏点 **「供应商」**
2. **「添加供应商」**，类别下拉可选：酒店/花艺、摄影摄像、化妆造型、餐饮、司仪、音乐/乐队、婚纱礼服、其他
3. 填合同金额、已付金额，卡片上会显示付款进度条，没结清会显示"还差 XX 元"

### 第 5 步：记支出，看有没有超

1. 顶栏点 **「预算支出」**
2. 顶部大卡片实时显示：总预算、使用进度条、剩余/超支金额
3. 点右上角 **「记一笔」**：
   - 支出类别（下拉选常用，也能自己输）
   - 说明、金额、日期
   - 可以关联到某家供应商
4. **超支提醒**：
   - 记账前，如果这笔会导致超支，弹窗下方会红字警告
   - 点「记录」时还会再弹 confirm 让你再确认一次
   - 确认入账后，顶部卡片变红，超支金额醒目显示
5. 左边是 **按类别汇总** 的条形图，右边是 **支出明细表**

## 核心设计思路

### 🔗 任务依赖怎么存的

用单独一张 `task_dependencies` 表（task_id, depends_on_id），多对多。

判断「能不能开始」：查这张表里该任务所有 depends_on_id 对应的任务是不是全是 `completed` 状态。

**三道防线防漏事**：
1. 加依赖时做 **循环依赖检测**（DFS 深搜，如果加了这条依赖能绕回自己就拒绝）
2. 切换状态时服务端 **再次校验**，不依赖前端
3. 前端看板上未解锁的任务卡片会显示 🔒 图标 + 还缺哪些任务

### 🪑 座位怎么安排才不吵架

> 你没想透的地方我是这么设计的，你可以后面再改：

1. **分组 + 桌次双层管理**。左侧两个面板：上面是分组（按人情关系分），下面是桌次（按物理座位分）
2. 每个宾客同时归属于一个分组 + 一桌，互不冲突
3. 统计时按两个维度：
   - 分组维度：男方亲戚多少人 → 方便确认有没有漏请长辈
   - 桌次维度：每桌坐了谁 → 防止桌超员，现场挤
4. **桌超员标红**：`已入座 > 容量` 时桌名旁边亮红「超员」，比单纯算席数更直观
5. 后续可以加：按分组一键排桌、同桌宾客的关系禁忌提示

### 💰 预算超支实时比

每笔支出写入 `expenses` 表。每次打开预算页或请求 `/api/weddings/[id]/budget` 时：
- 实时 `SUM(amount)` 算总支出，跟婚礼上存的 `budget_total` 做差
- 返回 `is_over_budget`、`overage_amount`
- 前端用这两个值控制颜色（红色/正常色）和进度条填充

**双重校验**：
- 记一笔前，前端先实时算"加了这笔会不会超"
- 提交时服务端 `validateBudget` 再独立算一次，返回 HTTP 202（will_over=true）或 201

## 技术栈

| 层 | 技术 | 说明 |
|----|-----|-----|
| 前端 | Svelte 4 + SvelteKit 2 | 自带 UI，不引额外框架 |
| 服务端 | SvelteKit + Node adapter | 数据全走 SSR + Server API |
| 数据库 | SQLite (better-sqlite3) | 手写原生 SQL，无 ORM |
| 测试 | Jest + ts-jest | 任务依赖 + 预算校验 |

## 自动化测试

覆盖两个最容易出 bug 的核心逻辑：

```bash
npm test
```

**任务依赖测试（15 个）**：
- 无依赖可开始、有依赖锁任务、前置完成解锁
- 状态切换校验（in_progress / completed 被阻止）
- 多层链依赖（A→B→C）、多前置（A+B→C）
- 自依赖拒绝、直接循环（A↔B）拒绝、长链循环（A→B→C→A）拒绝
- 非循环不误报、跨婚礼依赖拒绝、幂等依赖不报错
- 移除依赖任务恢复可启动

**预算校验测试（7 个）**：
- 空预算、单笔、多笔分类统计正确
- 超支判定（is_over_budget + overage_amount）
- validateBudget 预判（加这笔会不会超）
- 零预算直接超支、删除支出后预算恢复

## 数据库表结构

| 表 | 作用 |
|----|-----|
| weddings | 婚礼主表（新人、日期、预算） |
| tasks | 任务（标题、负责人、截止日、状态） |
| task_dependencies | 任务依赖（task_id, depends_on_id） |
| guest_groups | 宾客分组 |
| tables | 桌次（桌号、容量） |
| guests | 宾客（姓名、分组、桌次、到场、携伴） |
| suppliers | 供应商（类别、合同金额、已付） |
| expenses | 支出明细（类别、金额、关联供应商） |

完整 SQL 见 [src/lib/server/schema.sql](src/lib/server/schema.sql)
